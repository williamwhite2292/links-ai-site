# Blog publish checklist (Carl / content workflow)

**Status:** Ready for publishing workflow  
**Last updated:** 2026-03-15  
**Astro build:** Verified working ✓

This doc defines how approved blog content from the shared workspace becomes a published post on the site.

## Content collection

- **Location:** `src/content/blog/`
- **Format:** One `.md` file per post. Filename becomes the URL slug (e.g. `my-post-title.md` → `/insights/my-post-title`).

## Frontmatter (required)

Every post must have:

```yaml
title: "Post title"
description: "Short description for SEO and listing."
pubDate: 2026-03-15
author: Links AI
tags:
  - tag-one
  - tag-two
draft: false
```

Optional: `updatedDate`, `canonical`. Do not add `slug` (Astro derives it from the filename).

## “Approved and ready for Carl”

A post is ready for publish when (a) the file exists in the shared workspace at `content/blog/approved/{slug}.md`, and (b) Abby has marked it approved (no pending revisions). Only then should Carl copy it into the site repo.

## Publish steps (Carl)

1. Take the **approved** post from the shared workspace (`content/blog/approved/{slug}.md` or equivalent).
2. Copy the markdown body and frontmatter into a new file: `src/content/blog/{slug}.md`. Use a URL-friendly slug as the filename (e.g. `how-to-start-with-ai.md`).
3. Set `draft: false`. Ensure `title`, `description`, `pubDate`, and `author` are set.
4. Run `npm run build` to regenerate the site (sitemap is auto-generated by `@astrojs/sitemap`).
5. Deploy the `dist/` output.

## Current site structure
